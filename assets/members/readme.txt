Drop each member's photos here to replace the auto-generated placeholder.
PNG, JPG, and WEBP all work, and you can mix formats freely.

Name each photo with the member's short nickname, then a number: 1, 2, 3...
The app tries 1, then 2, then 3, and so on, stopping at the first number
that's missing — so you can add as many (or as few) photos as you like.

  bc = Bang Chan     (bc1.jpg, bc2.jpg, ...)
  lk = Lee Know       (lk1.jpg, lk2.jpg, ...)
  cb = Changbin       (cb1.jpg, cb2.jpg, ...)
  hj = Hyunjin        (hj1.jpg, hj2.jpg, ...)
  h  = Han            (h1.jpg, h2.jpg, ...)
  f  = Felix          (f1.jpg, f2.jpg, ...)
  s  = Seungmin       (s1.jpg, s2.jpg, ...)
  in = I.N            (in1.jpg, in2.jpg, ...)

If a member has 2 or more photos, the app shuffles between them every few
seconds, never showing the same one twice in a row. A member with only one
photo just shows that one, all the time. The WHOLE photo is always shown —
it's shrunk to fit if needed, but never stretched or cut off.

Portrait shots look best (roughly 3:4, e.g. 600x800), but any shape works.

If a member has none of these numbered photos, the app looks for one old-
style flat file as a fallback (PNG or JPG, preferred first):

  1_bang_chan.png   2_lee_know.png   3_changbin.png   4_hyunjin.png
  5_han.png         6_felix.png      7_seungmin.png   8_in.png

If NEITHER is found, the app just shows a clean placeholder with the
member's name — it never crashes over a missing picture.
